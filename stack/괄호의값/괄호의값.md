# 괄호의 값

## 생각할 부분

1. 덧셈과 곱셈 연산이 언제 진행되는가?

- 덧셈 : 숫자끼리 붙어있을 경우 실행 된다
  - 스택의 마지막에 숫자만 남았을 경우 더해야 함
  - 닫는 괄호를 만나 괄호 내부의 연산을 수행해야 할 경우 더해야 함
- 곱셈 : 괄호 내부의 숫자가 다 더해진 이후에 실행된다 = 괄호 내부에 숫자가 있다면 일단 더하고 마지막에 곱해야 한다

2. 숫자는 언제 생성되는가?

- 내가 돌고 있는 값과 짝이 맞는 값이 스택의 최상단에 위치해 있을 때 둘이 만나면서 생성됨

3. 괄호의 오류는 언제 알 수 있는가? (중요)

- 닫는 괄호를 만났을 때, 스택에서 pop한 값이 숫자나 여는 괄호가 아니라면 오류
- 이미 짝이 맞았던 애들은 다 숫자로 연산 돼서 스택에 push 됐을 것
- 여는 괄호는 해당 연산들을 감싸는 괄호를 의미
- 그 외의 값이 나온다는 건 짝이 맞지 않는다는 의미

사칙 연산을 스택으로 푸는 것을 생각해보면 이해하기 쉽다

## 스택으로 푸는 사칙연산

1. 여는 괄호는 무조건 스택에 push
2. 곱셈 / 나눗셈은 무조건 곧바로 연산한 뒤 결과값을 스택에 push
3. 덧셈 / 뺄셈은 연산하지 않고 바로 스택에 push
4. 닫는 괄호를 만나면 여는 괄호를 다시 만날 때까지 내부 연산 모두 실행 및 결과값 push

위의 기본 과정에서 +a로,

- 연산 값 : 괄호의 짝이 맞는 순간 생성됨
- 곱셈 : 괄호 내부의 덧셈이 모두 실행된 후, 하나의 연산 값이 나오면 실행 됨
  이 2가지를 적용하면 된다
